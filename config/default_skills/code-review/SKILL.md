# Code Review

## 1. 拉取代码

git pull 最新代码，检查分支名和 diff 范围。

## 2. 逐文件检查

关注以下维度：

- **正确性**：逻辑是否正确，边界情况是否处理
- **规范**：命名、格式、docstring、type hints
- **复杂度**：函数是否过长，是否有拆分空间
- **测试**：是否有测试覆盖，覆盖率是否足够
- **安全**：是否有硬编码密钥、SQL 注入等风险

## 3. 提交报告

列出问题（按严重程度排序），给出具体修改建议。问题用代码行号定位。
