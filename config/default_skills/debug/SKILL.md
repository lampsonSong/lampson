## debug 技能

### 描述
帮助用户调试代码、排查错误和修复问题。

### 步骤
1. 读取报错信息和相关代码（用 file_read 读取出错的文件）
2. 分析错误类型和根本原因
3. 提出修复方案，说明为何出错
4. 用 file_write 或直接给出修改后的代码
5. 用 shell 执行修改后的代码验证修复效果
6. 如果问题未解决，继续分析（最多 3 轮尝试）

### 调试策略
- 优先检查 traceback 的最后一行（直接原因）
- 向上追溯调用栈找根本原因
- 检查变量类型和值是否符合预期
- 必要时插入 print/logging 语句辅助排查

### 注意事项
- 修改代码前先备份或确认用户已有 git 记录
- 不要只修复症状，要找到根本原因
- 如果错误复杂，分步骤向用户解释
