# Debug

严格按阶段执行，不跳步。

## Phase 1 — 复现

拿到明确的报错信息或异常行为。没有复现就不要改代码。

## Phase 2 — 定位

- 优先看 traceback 最后一行（直接原因）
- 向上追溯调用栈找根本原因
- 检查变量类型和值是否符合预期
- 必要时插入 print/logging 辅助排查
- 只做只读探测（ls、cat、which、env），不要盲目全盘搜索

## Phase 3 — 修复

- 在项目源码中直接修复，不留临时补丁
- 修复后跑验证，确认 bug 不再复现

## Phase 4 — 清理

- 清理调试过程中产生的临时文件
- 如果修复逻辑有通用性，考虑沉淀到 skill

## 禁止

- 禁止用临时脚本（/tmp/xxx.py）绕过问题
- 禁止用 workaround、临时 patch 掩盖症状
- 必须找到根本原因，用最彻底的方式修复
