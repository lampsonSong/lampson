# Grill

**在动手之前，先反复追问直到理解你的意图。**

参考自 Matt Pocock 的 grill-me skill。核心理念：没有人一开始就完全清楚自己要什么。通过追问把模糊的想法变成清晰的设计。

## 工作流

1. 用户提出一个想法或需求
2. 逐条追问每个方面的细节，每次只问一个问题
3. 如果问题可以通过探索代码/项目找到答案，先自己探索再问
4. 对每个问题给出推荐答案
5. 沿决策树向下走，逐个解决依赖关系
6. 所有决策点确认后，总结为执行方案

## 触发时机

- 用户说"帮我做一个xxx"，但细节不明确
- 新功能的实现方案有多种选择
- 改动涉及多个模块，影响面不清晰
- 用户主动要求"先理清楚"

## 追问维度

- **目标**：最终要达成什么效果？
- **范围**：包括什么，不包括什么？
- **优先级**：哪些必须做，哪些可以后做？
- **约束**：技术限制、时间限制、兼容性要求？
- **验收**：怎么算做完了？

## 格式

每个问题：
> **Q**: [问题]
> **推荐**: [我的建议]
> 
> 用户回答后 → 下一个问题

## 规则

- 不要一次性抛出所有问题，逐个来
- 如果能通过看代码/项目自己回答，不要问
- 问到用户说"够了"或"就这样"时停止
- 不确定时就问，不要猜
