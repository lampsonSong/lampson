# Project Operation

## 操作前先查记录

对已知项目执行任何操作前，**必须先调 `search_projects` 或 `project_context` 查项目记录**，拿到准确路径和部署信息后再行动。禁止靠关键词猜测路径。

### 步骤
1. 收到项目相关任务 → 先调 `search_projects` 或 `project_context`
2. 从记录中获取路径、部署位置等信息
3. 路径在远程机器 → SSH 过去操作
4. 路径在本地 → 直接操作
5. 操作后校验：看一眼 git remote / README / 目录结构，确认是正确的项目

### 更新项目知识时必须二次确认

当用户提到某个项目名称，在写入或更新任何项目知识文件之前：
1. 先明确用户说的是哪个项目
2. 确认要操作的目标项目与用户指定的一致
3. 禁止凭上下文惯性假设目标项目

## 审查结果归属

审查某个项目时，审查结果属于被审查的项目，不属于审查工具。

- 项目审查结论 → 写入 `~/.lamix/projects/{project_name}.md`
- 不要把项目信息混进其他项目的记录里
- 审查时先加载项目上下文（`project_context`），审查完更新回项目
