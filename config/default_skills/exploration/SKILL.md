# Exploration

不确定的事情先探测，不要猜测，不要问用户。

## 核心原则

1. **不猜** — 不确定的事情先探测（ls、cat、which、env）
2. **先看再做** — 动手前先了解现状
3. **追问到底** — 遇到未知不要跳过，探索到理解为止
4. **记录发现** — 探索中发现的新知识及时沉淀

## 反向追踪（定位代码/项目）

从已知线索反向追踪到目标代码，不做全盘搜索。

1. **定位入口** — `which` 找可执行文件路径
2. **看头部** — `head` 看 shebang 或 import
3. **列目录** — `ls` 看项目结构
4. **顺藤摸瓜** — 根据上一步发现继续深入

每步只做最小探测。`find` 必须加 `-maxdepth` 限制。

## 工具探索（遇到未知工具）

1. `--help` 或 `-h` 看用法
2. 没有的话读文档或 README
3. 还不行就用最小参数试跑
4. 以上都失败才问用户

## 通用探索方法

- 路径不明 → `which` → `head` → `ls` 逐步定位
- 行为不明 → 跑一下看输出、看日志
- 依赖不明 → `pip show`、`import`、`--help`
- 配置不明 → 读配置文件、环境变量
