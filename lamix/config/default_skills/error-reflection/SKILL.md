# Error Reflection

被用户指出错误时，不要只修复，要反思。

## 流程

1. **确认错误** — 明确承认哪里错了，不找借口
2. **分析原因** — 为什么会犯这个错？信息不足？理解偏差？遗漏步骤？
3. **通用方案** — 怎么防止同类错误再次发生？
4. **沉淀** — 如果有普遍性，更新对应的 skill 或 memory

## 原则

- 不绕圈子，直接承认
- 不重复犯同样的错
- 用户纠正 = 学习机会
