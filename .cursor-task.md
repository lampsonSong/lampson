# 任务：progress_callback 改为异步队列模式

## 背景

当前 Lampson 的 progress_callback 是同步的：每完成一个工具调用，`_on_tool_progress` 直接调用飞书 API 发/更新卡片。这有两个问题：
1. **阻塞工具循环** — 飞书 API 调用（发卡片、更新卡片）是网络请求，会阻塞 LLM 工具循环
2. **没有节流** — 如果工具调用很快完成，可能短时间内打很多飞书 API

## 参考：Hermes 的做法

Hermes 在 `gateway/run.py` 的处理方式（简化版）：

```python
# 1. 用 queue.Queue 作为缓冲
progress_queue = queue.Queue()

# 2. progress_callback 只往队列里塞数据（不阻塞主循环）
def progress_callback(event_type, tool_name=None, preview=None, ...):
    progress_queue.put(msg)

# 3. 后台 async 任务从队列取数据，批量更新飞书卡片
async def send_progress_messages():
    progress_lines = []      # 累积的工具行
    progress_msg_id = None   # 已发送卡片的 message_id
    _last_edit_ts = 0.0
    _PROGRESS_EDIT_INTERVAL = 1.5  # 最少间隔 1.5 秒

    while True:
        raw = progress_queue.get_nowait()  # 非阻塞取
        progress_lines.append(msg)

        # 节流：1.5s 内只编辑一次
        _remaining = _PROGRESS_EDIT_INTERVAL - (now - _last_edit_ts)
        if _remaining > 0:
            await asyncio.sleep(_remaining)
            continue  # 回来再取积攒的消息，一次性更新

        if progress_msg_id is None:
            # 第一次：发新卡片
            result = adapter.send(...)
            progress_msg_id = result.message_id
        else:
            # 后续：更新卡片
            adapter.edit_message(progress_msg_id, ...)

        _last_edit_ts = time.monotonic()
```

## 需要改的文件

### 1. `src/core/agent.py` — `_on_tool_progress`

当前代码（同步直接调 callback）：
```python
def _on_tool_progress(self, round_num, tool_name, args, result):
    if not self.progress_callback:
        return
    self.progress_callback({
        "type": "tool_progress",
        "round": round_num,
        "tool": tool_name,
        "args_preview": args_preview,
        "result_preview": result_preview,
    })
```

这个不用改，callback 本身已经是往队列里塞数据的轻量操作，不会阻塞。但如果 callback 还是比较重，可以让它只做 `queue.put()`。

### 2. `src/feishu/listener.py` — `_handle_message` 里的 `_progress_cb`

**这是主要改动点。** 当前是同步的：

```python
def _progress_cb(event):
    # 直接调飞书 API，阻塞主循环
    _prog_state["lines"].append(...)
    if msg_id is None:
        _prog_state["msg_id"] = self._send_progress_card(chat_id, lines)  # 阻塞！
    else:
        self._update_progress_card(msg_id, lines)  # 阻塞！
```

改为：

```python
import queue
import threading

def _progress_cb(event):
    # 只往队列里塞数据，不阻塞
    _progress_queue.put(event)

# 后台线程负责消费队列、调飞书 API
def _progress_worker():
    progress_lines = []
    progress_msg_id = None
    last_update_ts = 0.0
    UPDATE_INTERVAL = 1.5  # 秒

    while True:
        try:
            event = _progress_queue.get(timeout=0.5)
        except queue.Empty:
            # 检查是否该结束
            if _done_event.is_set():
                # 最后一次更新（标记完成）
                if progress_msg_id and progress_lines:
                    self._update_progress_card(progress_msg_id, progress_lines, finished=True)
                return
            continue

        if not isinstance(event, dict) or event.get("type") != "tool_progress":
            continue

        # 解析事件，追加到 lines
        ...
        progress_lines.append(f"...")

        # 节流
        now = time.monotonic()
        if now - last_update_ts < UPDATE_INTERVAL:
            continue  # 等下次积攒更多再更新

        # 发/更新卡片
        if progress_msg_id is None:
            progress_msg_id = self._send_progress_card(chat_id, progress_lines)
        else:
            self._update_progress_card(progress_msg_id, progress_lines)
        last_update_ts = time.monotonic()

# 启动后台线程
_done_event = threading.Event()
_progress_queue = queue.Queue()
worker_thread = threading.Thread(target=_progress_worker, daemon=True)
worker_thread.start()

# ... handle_input ...

# 结束时
_done_event.set()
worker_thread.join(timeout=3)
```

### 3. 节流策略

- 最少间隔 1.5 秒更新一次卡片（和 Hermes 一样）
- 队列里积攒的消息会在下次更新时一起显示
- 工具循环结束后，设置 `_done_event`，worker 线程做最后一次"已完成"更新

## 约束

- 不要改 `agent.py` 的 `_on_tool_progress` 接口，它传的 dict 格式保持不变
- 不要改 `FeishuClient` 已有的方法
- 不要改 `client.py` 里的 `update_message` 和 `send_card` 签名
- 用 `threading` 而不是 `asyncio`（Lampson 是同步架构，不像 Hermes 用 asyncio）
- 后台线程设为 daemon=True，防止进程挂住
- 保留已有的 `_send_reply`、`_send_progress_card`、`_update_progress_card`、`_make_progress_card` 方法
- 跑 `pytest tests/ -q` 确认 187 个测试全通过
