# Cursor Task: 修 should_trigger + review session 模块

## 1. 修改 should_trigger（主要工作）

**文件**：`src/core/compaction.py`

**目标**：把硬编码的触发条件改为从 config 读取的百分比。

**当前代码**（约第 56-64 行）：
```python
def should_trigger(self, message_count: int, estimated_tokens: int, stop_reason: str | None) -> bool:
    if stop_reason not in STOP_REASONS:
        return False
    if message_count < 150:
        return False
    if estimated_tokens < 60_000:
        return False
    return True
```

**改为**（设计文档要求）：
```python
def should_trigger(self, message_count: int, estimated_tokens: int, stop_reason: str | None) -> bool:
    if stop_reason not in STOP_REASONS:
        return False
    # 基于百分比触发，不依赖硬编码条数
    threshold_tokens = self.context_window * self.trigger_threshold
    return estimated_tokens >= threshold_tokens
```

其中 `self.trigger_threshold` 和 `self.context_window` 从 config.yaml 的 compaction 段读取。

config.yaml 已有字段：
```yaml
compaction:
  context_window: 131072  # GLM-5.1 = 128K
  trigger_threshold: 0.8  # 80%
```

注意：不需要 `message_count` 参数了，触发只看 token 占比。
