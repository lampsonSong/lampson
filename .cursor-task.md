## 具体改动

### 1. memory_show 项目标题优化（skills_tools.py 第213行附近）

**问题**：`md_file.stem`（如 `2026-04-25`）作为项目名不友好

**修复**：优先读文件第一行作为标题（匹配 `# 标题` 格式），如果读不到或为空才用 stem

```python
# 原来：
projects.append(f"### {md_file.stem}\n\n{preview}")

# 改为：
title = md_file.stem  # 默认用文件名
try:
    first_line = content.split('\n')[0].strip()
    if first_line.startswith('# '):
        title = first_line[2:].strip()
except (OSError, IndexError):
    pass
projects.append(f"### {title}\n\n{preview}")
```

### 2. memory_show 子目录路径问题

**问题**：`rglob` 递归时 `md_file.stem` 会包含路径分隔符（如 `foo/bar`），导致标题变成 `foo/bar`

**修复**：用 `relative_to` 获得相对路径，再用 `stem`

```python
# 原来：
for md_file in sorted(PROJECTS_DIR.rglob("*.md")):

# 改为：
for md_file in sorted(PROJECTS_DIR.rglob("*.md")):
    rel_path = md_file.relative_to(PROJECTS_DIR)
    # rel_path 形如 "foo/bar.md"，stem 只有 "bar"
    # 但如果文件名是 bar.md，stem 就是 "bar"
    # 需要的是相对路径的友好展示
```

**实际上**：`stem` 本身就不包含路径，所以问题在于当 `PROJECTS_DIR` 下有子目录如 `projects/foo/bar.md` 时，`md_file.stem` 仍是 `bar`。真正的问题是标题可能重复（多个子目录都有 bar.md）。可以用 `rel_path` 的父目录前缀来区分：

```python
rel_path = md_file.relative_to(PROJECTS_DIR)
if rel_path.parent != Path('.'):
    # 有子目录，在标题前加父目录名
    parent_name = str(rel_path.parent).replace('/', ' > ')
    title = f"{parent_name} > {title}"
```
